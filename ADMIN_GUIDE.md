# 京吹学报 - 在线文章管理系统使用指南

欢迎使用京吹学报的在线文章管理系统！这个系统让你可以直接在网页上编写和发布文章，无需在 GitHub 后台操作。

## 📝 访问地址

**正式环境**: https://hibikilogy.github.io/admin.html

## 🚀 快速开始

### 第一步：创建 GitHub Personal Access Token

1. 访问 GitHub Token 创建页面：https://github.com/settings/tokens/new?scopes=repo
2. 填写以下信息：
   - **Note**: `京吹学报编辑器` (或任何你喜欢的名称)
   - **Expiration**: 选择过期时间（建议选择 90 天或自定义）
   - **Select scopes**: 确保勾选了 `repo` (完整的仓库访问权限)
3. 点击页面底部的 "Generate token" 按钮
4. **重要**: 复制生成的 Token（格式类似 `ghp_xxxxxxxxxxxx`），这个 Token 只会显示一次！

> ⚠️ **安全提示**: Token 相当于你的密码，请妥善保管，不要分享给他人。

### 第二步：登录编辑器

1. 访问 https://hibikilogy.github.io/admin.html
2. 在登录页面输入你的 GitHub Token
3. 点击"登录"按钮
4. 登录成功后，Token 会保存在浏览器本地，下次访问无需重新输入

## ✍️ 编写文章

### 创建新文章

1. 点击工具栏的"新建文章"按钮
2. 填写文章元数据：
   - **标题** (必填): 文章的主标题
   - **作者** (必填): 你的名字或笔名
   - **日期** (必填): 文章发布日期，默认为今天
   - **副标题** (可选): 文章的副标题或简短描述
   - **标签** (可选): 用逗号分隔，例如：`吹学, 分析, 评论`
   - **原文链接** (可选): 如果是转载或有原文链接
   - **布局**: 通常选择 `post`

3. 在编辑器中编写文章内容（支持 Markdown 语法）

### 使用 Markdown 编辑器

编辑器工具栏提供了常用的 Markdown 格式化功能：

- **B** - 粗体
- **I** - 斜体
- **H** - 标题
- **"** - 引用
- **•** - 无序列表
- **1.** - 有序列表
- **🔗** - 插入链接
- **🖼️** - 插入图片
- **📊** - 插入表格
- **👁️** - 预览
- **↔️** - 分屏预览
- **⛶** - 全屏模式

### 插入图片

有两种方式插入图片：

**方式 1: 使用图床**
```markdown
![图片描述](https://i.loli.net/2020/03/13/xxxxx.png)
```

**方式 2: 使用仓库图片**
```markdown
![图片描述](/images/2024/2024-12-01/image.jpg)
```

> 💡 **提示**: 建议使用图床（如 SM.MS、路过图床等），这样图片加载更快。

### 保存草稿

- 编辑器会自动保存草稿到浏览器本地存储
- 你也可以点击"保存文章"按钮手动保存
- 即使关闭浏览器，草稿也会保留

## 📤 发布文章

### 发布到 GitHub

1. 确保所有必填字段都已填写
2. 点击工具栏的"发布到 GitHub"按钮
3. 系统会自动：
   - 生成符合规范的文件名（格式：`YYYY-MM-DD-标题.md`）
   - 创建 Front Matter（文章元数据）
   - 将文章保存到 `_posts/年份/` 目录
   - 提交到 GitHub 仓库

4. 发布成功后，文章会出现在"最近文章"列表中
5. 等待几分钟，GitHub Pages 会自动构建并发布网站

> ⏱️ **构建时间**: 通常需要 2-5 分钟，你可以在仓库的 Actions 页面查看构建进度。

## 📝 编辑现有文章

1. 在页面底部的"最近文章"列表中找到要编辑的文章
2. 点击文章旁边的"编辑"按钮
3. 系统会自动加载文章内容和元数据
4. 修改后点击"发布到 GitHub"更新文章

## 📋 文章命名规范

文章文件名会自动生成，格式为：`YYYY-MM-DD-标题.md`

例如：
- 标题：`黄前久美子的两个身体`
- 日期：`2024-06-03`
- 生成的文件名：`2024-06-03-黄前久美子的两个身体.md`
- 保存路径：`_posts/2024/2024-06-03-黄前久美子的两个身体.md`

## 📐 Front Matter 格式

系统会自动生成 Front Matter，格式如下：

```yaml
---
layout: post
title: "文章标题"
subtitle: "副标题"
author: "作者名"
original: "原文链接"
header-img: "https://i.loli.net/2020/03/13/Kk83Nb1DUaYOi6j.png"
catalog: true
tags:
    - 标签1
    - 标签2
    - 标签3
---
```

## 🔧 常见问题

### Q: Token 过期了怎么办？

A: 重新创建一个新的 Token，然后在登录页面输入新 Token。

### Q: 发布失败怎么办？

A: 检查以下几点：
1. Token 是否有效且具有 `repo` 权限
2. 必填字段是否都已填写
3. 网络连接是否正常
4. 查看浏览器控制台是否有错误信息

### Q: 文章发布后多久能看到？

A: GitHub Pages 通常需要 2-5 分钟构建和部署。你可以在仓库的 Actions 页面查看构建状态。

### Q: 可以删除文章吗？

A: 目前编辑器不支持删除功能，需要在 GitHub 仓库中手动删除文件。

### Q: 图片上传到哪里？

A: 编辑器不提供图片上传功能，建议使用图床服务：
- SM.MS: https://sm.ms/
- 路过图床: https://imgtu.com/
- ImgURL: https://imgurl.org/

### Q: 如何预览文章效果？

A: 点击编辑器工具栏的"预览"或"分屏预览"按钮，可以实时查看 Markdown 渲染效果。

### Q: 草稿保存在哪里？

A: 草稿保存在浏览器的本地存储中。如果清除浏览器数据，草稿会丢失。建议定期发布或手动备份重要内容。

## 🔒 安全建议

1. **不要分享你的 Token**: Token 相当于密码，可以访问你的 GitHub 仓库
2. **定期更换 Token**: 建议每 90 天更换一次
3. **使用 HTTPS**: 确保访问的是 `https://` 开头的网址
4. **退出登录**: 在公共电脑使用后，记得点击"退出"按钮
5. **撤销 Token**: 如果 Token 泄露，立即在 GitHub 设置中撤销：https://github.com/settings/tokens
