# 📋 项目重构方案总结

## 🎯 核心问题

当前京吹学报项目存在以下主要问题：

1. **文件组织混乱** - 15+ 文件散落根目录，难以维护
2. **图片管理无序** - 日期、中文、英文命名混用
3. **技术债务积累** - Travis CI 废弃、Grunt 无配置、缺少依赖管理
4. **文档不完善** - 开发文档缺失，贡献指南分散
5. **命名不一致** - 文章、图片、配置命名规范不统一

## 💡 解决方案

### 方案概览

采用**渐进式重构**，分 5 个阶段执行：

```
阶段 1: 基础重构 → 创建新目录结构，迁移文件
阶段 2: 标准化命名 → 统一文件命名规范
阶段 3: 现代化工具链 → 添加 npm、prettier、自动化脚本
阶段 4: 更新配置 → 优化 Jekyll 和构建配置
阶段 5: 自动化工具 → 图片优化、路径更新脚本
```

### 新目录结构

```
hibikilogy.github.io/
├── content/          # 📝 内容（文章、图片、页面）

```

## ⚠️ 风险与缓解

### 主要风险

1. **链接失效** - 内部链接可能失效
   - 缓解：使用脚本批量更新
   - 缓解：保持 URL 结构不变

2. **图片路径错误** - 大量图片引用需要修正
   - 缓解：自动化路径更新脚本
   - 缓解：充分测试

3. **SEO 影响** - URL 变化可能影响搜索排名
   - 缓解：保持 permalink 不变
   - 缓解：添加 301 重定向

### 回滚方案

如果出现问题，可以立即回滚：

```bash
bash tools/migration/rollback.sh backup_YYYYMMDD_HHMMSS
```
